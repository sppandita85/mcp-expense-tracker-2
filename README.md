# Expense Tracker MCP Server

A remote MCP (Model Context Protocol) server for tracking expenses, built with
[FastMCP](https://github.com/jlowin/fastmcp) and designed to run on
[FastMCP Cloud](https://fastmcp.cloud).

Storage is **in-memory** — a plain Python list. Data resets whenever the
server process restarts or is redeployed. There is no database.

## Tools

| Tool | Signature | Description |
|---|---|---|
| `add_expense` | `add_expense(amount: float, category: str, date: str, note: str = "")` | Records a single expense. Returns the stored record (including its id). |
| `get_monthly_summary` | `get_monthly_summary(month: str)` | Returns expenses for that month, grouped by category, as `{category: {"total": ..., "count": ...}}`. |
| `get_monthly_total` | `get_monthly_total(month: str)` | Returns the total amount spent in that month. |

`date` and `month` are free-text (e.g. `"2026-07-21"`, `"July 21, 2026"`,
`"07/21/2026"`, `"2026-07"`). The server parses them leniently
(`python-dateutil`) and matches on year + month; if a value can't be parsed
it falls back to a plain substring match instead of erroring out.

## Run and test locally

```bash
pip install -r requirements.txt
python server.py
```

This serves the MCP endpoint at `http://localhost:8000/mcp` over HTTP.

Smoke-test it with the official MCP inspector:

```bash
npx @modelcontextprotocol/inspector
```

Point it at `http://localhost:8000/mcp` (Streamable HTTP transport) and call
`add_expense`, `get_monthly_summary`, and `get_monthly_total` directly.

Alternatively, from Claude Code:

```bash
claude mcp add --transport http expense-tracker http://localhost:8000/mcp
```

## Deploy to FastMCP Cloud

1. Push this repo to GitHub (already done if you're reading this from the
   repo — see below).
2. Go to [fastmcp.cloud](https://fastmcp.cloud) and sign in with GitHub.
3. Authorize FastMCP Cloud to access this repository (`mcp-expense-tracker`).
4. Create a new project from the repo. FastMCP Cloud auto-detects the
   entrypoint by finding the `FastMCP` instance (`mcp`) in `server.py` — no
   extra config needed.
5. Deploy. FastMCP Cloud will build and host the server, then give you a
   URL of the form:

   ```
   https://<your-project-name>.fastmcp.app/mcp
   ```

   That URL is generated by the platform at deploy time — copy it from the
   FastMCP Cloud dashboard once the deploy finishes.

## Add it as a custom connector in Claude.ai

1. Open [claude.ai](https://claude.ai) and go to **Settings → Connectors**.
2. Click **Add custom connector**.
3. Paste the `/mcp` URL from FastMCP Cloud (step 5 above).
4. Give it a name, e.g. "Expense Tracker", and save.
5. In a chat, enable the connector via the tools/connector picker, then try:
   - "Add a $12 lunch expense today in the Food category"
   - "What's my total spend this month?"
   - "Break down this month's expenses by category"

## Notes

- Data is **not persisted** — every redeploy or restart clears all expenses.
- `amount` must be a positive number; `add_expense` raises an error otherwise.
